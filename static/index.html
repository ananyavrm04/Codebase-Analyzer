How it works
Architecture
GitHub URL → Download & Parse → AST Chunk → Embed → FAISS Index → Search / Q&A

🌳 AST-Based Chunking

Code is parsed into functions and classes using Python's AST module — not arbitrary character splits. Each chunk is a complete semantic unit. Falls back to sliding window for files with syntax errors.

⚡ FAISS HNSW Index

HNSW (Hierarchical Navigable Small World) provides sub-10ms vector search with high recall. Chosen over IVF because codebase repos produce < 100k chunks — HNSW is faster and needs no training step at this scale.

🔄 Configurable Embeddings

Default: all-MiniLM-L6-v2 via sentence-transformers (local, free, 384-dim). Switch to OpenAI text-embedding-3-small via env var — zero code changes. Plug-and-play model swapping.

🤖 LLM Fallback Chain

Together AI (primary) → Anthropic Claude (fallback). The Q&A endpoint retrieves context via FAISS, then passes it to the LLM with structured prompts. If the primary fails, the fallback fires automatically.

⏱️ Async Indexing

POST /index returns immediately with a job_id. Indexing runs as a background task — download, parse, chunk, embed, build index. Poll /index/{id}/status for progress. No request timeouts on large repos.

📊 Sub-200ms Latency

FAISS HNSW search completes in < 10ms. Full end-to-end (embed query + search + return results) in < 200ms. Tested on repositories up to 80MB with thousands of functions.
